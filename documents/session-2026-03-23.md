# Session Notes — March 23, 2026

## Customizable Disciplines

Added a preferences system so users can choose which items appear in their daily checklist.

- New localStorage key `exodus40lite-prefs` stores per-item on/off state
- All existing items default to on; items marked `optional: true` default to off
- `shouldShow()` now checks preferences before frequency logic
- "Customize your disciplines" expandable panel below the checklist with iOS-style toggle switches grouped by category
- Added "Take a cold shower" as the first optional discipline (Asceticism category)

## Password Reset

Full forgot/reset password flow with SMTP email delivery.

### Backend
- `api/forgot-password.php` — accepts email, generates 64-char hex token (1-hour expiry), sends reset link via SMTP. Always returns success to prevent email enumeration.
- `api/reset-password.php` — validates token, updates password hash, invalidates all existing login tokens for the user.
- `password_resets` table added to schema and migrated to production DB.

### Frontend
- "Forgot password?" link in the sign-in form (hidden during register mode)
- Clicking it swaps login fields for an email-only form; "Back to sign in" returns to login
- On page load, `checkResetToken()` detects `?reset=TOKEN` in the URL, cleans it, and shows a new password + confirm form
- On success, shows a "Sign in" button that returns to the login form

### CSS fix
- `.forgot-container[hidden] { display: none; }` added because `display: flex` was overriding the `hidden` attribute, causing the forgot password inputs to show prematurely.
